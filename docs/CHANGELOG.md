## [Unreleased]

- **Feat: `saveSchema` game-state persistence (§7 backlog Tier 3, v4.35.0)**: new engine module `SaveGame` (`src/engine/SaveGame.js`) — browser-guarded (`typeof localStorage === 'undefined'` no-op, Node test env) persistence under one global key (`SaveGame.KEY`). New Event Graph actions `SaveGame`/`LoadGame` (`params: {variables?: string[], inventory?: boolean}`, inline schema — no separate catalog asset, same convention as `PlaySound`/`pathFollower.interpolation`). `save()` snapshots listed `scene.eventGraphRuntime` variables and/or `scene.inventory.list()` to `localStorage`; `load()` restores them (adds inventory counts onto whatever's already in the bag — no bulk-clear on `Inventory`, out of scope). Implements "across sessions" literally (browser tab close/reopen); "across levels" is supported by the same one-global-key mechanism but not exercised yet — level-switching itself needs the still-unregistered `LoadLevel` Event Graph action. Tests: `tests/engine/SaveGame.test.js` (11 cases — browser guards, variable/inventory round-trip, corrupted-JSON handling, missing-runtime no-op, clear()), 1 new end-to-end in `tests/engine/GameEngine.integration.test.js` (save on one GameEngine instance, load on a second, confirms cross-instance persistence via the shared `localStorage` mock); full suite 394/395 green (1 pre-existing unrelated flake, see `GameEngine.integration.test.js`'s rAF-timing dt test).
