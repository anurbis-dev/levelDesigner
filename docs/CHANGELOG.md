## [Unreleased]

- **Feat: HUD Canvas editor authoring UI (v4.36.0)**: fills the "no editor UI yet" gap left by v4.33.0's HUD Canvas runtime. New level-scope dock panel **Canvases** (`src/ui/canvas-hud/CanvasHudPanel.js` + pure model `CanvasHudModel.js`, registered in `DockConstants.js`/`DockPanelFactory.js` alongside Dialogues/Items) — canvases → widgets → widget form, same commit/undo pattern as `DialoguesPanel`/`ItemsPanel` (`HistoryManager.setCanvasesProvider`/`_snapshotCanvases`, `HistoryOperations.restoreCanvasesFromHistory`, `canvasesRevision` state key). Widget form: type/anchor pickers, offset/size number fields, static text/imgSrc, and a data-binding section (variable or inventory-count, with item picked from a dropdown, variable/custom-event names offered via `<datalist>` suggestions scraped from the level's Event Graph nodes — not a hard dropdown, since those names may not exist yet when first wiring a widget). Camera's `canvasIds` property switches from a hand-typed comma list (`stringList`) to a new **`idMultiSelect`** ComponentPropertySchema field kind — checkbox group dynamically sourced from `level.canvases` (`AssetComponentDetailsPanel._dynamicIdOptions()`/`_fieldHtml()`/`_bind()`), mirroring the existing `assetRef` dynamic-dropdown pattern but for level-scope id catalogs instead of the asset library. Tests: `tests/CanvasHudModel.test.js` (7 cases — canvas/widget CRUD, id generation, variable/custom-event name scraping); browser-verified: Canvases panel add/edit canvas+widget end-to-end, camera's HUD Canvases checkbox list renders live options and patches `component.properties.canvasIds` on toggle.
