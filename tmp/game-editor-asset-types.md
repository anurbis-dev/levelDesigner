# Типы ассетов для универсального редактора игр

## Ключевая архитектурная развилка

Прежде чем плодить типы — стоит разделить два разных понятия, которые иначе смешиваются и дают комбинаторный взрыв сущностей:

1. **Asset-типы (контент/ресурсы)** — то, что импортируется или создаётся как файл: картинка, атлас, звук, уровень, диалог, префаб.
2. **Component-типы (поведение)** — то, что **навешивается на Actor**, чтобы дать ему функциональность: коллизия, триггер, анимация трансформа, инвентарный предмет, интерактивность, урон и т.д.

Твой `Actor` уже описан как "контейнер общего типа" — это правильный путь. Вместо того чтобы создавать `Door`, `Lever`, `Chest`, `Vehicle`, `NPC` как отдельные **типы ассетов**, лучше сделать их **префабами Actor'а с определённым набором компонентов**. Тогда список типов ниже делится на "то, что действительно новый тип ресурса" и "то, что это просто компонент/поведение Actor'а" — я это пометил.

---

## 1. Уже определено тобой

| Тип | Комментарий |
|---|---|
| `Camera` | ок как отдельный тип — у неё нет визуала, только параметры (FOV/zoom, follow-target, границы, слои рендера) |
| `Level` | контейнер сцены |
| `Actor` | универсальный контейнер: визуал + коллизии + триггеры + анимация трансформа + UI/спрайт-анимация |
| `Image` | сырой растровый ресурс |
| `Image Atlas` | авто-упаковка картинок — сюда же стоит сразу заложить **Sprite Sheet / Frame Animation** (нарезка атласа на кадры с таймингом) — иначе Actor не сможет анимироваться спрайтами |
| `Volume` | зона-триггер произвольной формы + визуальные эффекты (блюр, цветфильтр, затенение) |

## 2. Content-ассеты, которых не хватает

### Визуал / рендер
- **Sprite Animation Clip** — последовательность кадров из Atlas + тайминги + loop-режим (once/loop/ping-pong). Без этого "анимация UI для спрайтовой анимации" в Actor не на чём собирать.
- **Tileset + Tilemap** — критично почти для любого 2D-жанра (платформер, top-down, метроидвания). Tileset = сетка картинок с правилами автотайлинга (авто-подбор углов/стыков), Tilemap = слой уровня, застроенный этими тайлами. Без этого застройка больших уровней вручную через Image крайне медленная.
- **Nine-Slice Sprite** — картинка с растягиваемыми краями, нужна для UI-рамок (окна диалогов, полоски здоровья, кнопки) и для масштабируемых декораций.
- **Font / Text Style** — шрифт + параметры (размер, обводка, тень) для UI и диалогов.
- **Particle Effect** — VFX-ассет (спрайт-частицы, эмиттер, гравитация/скорость/цвет по времени жизни). Нужен для магии, взрывов, дыма, следов от транспорта.
- **Material / Shader Preset** — переиспользуемый набор фильтров (по сути параметризация того, что уже задумано в Volume — блюр/цвет/затенение), чтобы не настраивать их вручную в каждом Volume заново.
- **Light** — источник света (точечный/направленный/зона), для 2D — с мягкими тенями поверх тайлмапа.
- **Parallax Layer / Background** — фоновый слой с независимой скоростью скролла, нужен почти в любом сайд-скроллере.

### Аудио
- **Sound Effect (SFX)** — короткий звук, привязываемый к событиям (шаг, удар, подбор предмета).
- **Music Track** — с поддержкой кроссфейда/luup-точек.
- **Audio Zone** — по сути специализация `Volume`: зона с амбиентным звуком/музыкой, которая включается при входе игрока.

### Данные / система
- **Dialogue Graph** — граф реплик с ветвлением, условиями показа веток (по флагам/квестам/инвентарю) и действиями на выбор (дать предмет, установить флаг, начать квест). Нужен для RPG, квестов, визуальных новелл, adventure.
- **Quest / Objective** — определение задания: список шагов, условия завершения, награда, связь с Dialogue Graph и Trigger-зонами.
- **Item Definition** — данные предмета (иконка = Image, название, тип: ключевой/расходуемый/экипировка, стек, эффект при использовании). Отдельно от Actor'а-подборщика на уровне (тот — просто Actor с компонентом Pickup, ссылающийся на Item Definition).
- **Inventory Schema** — структура слотов/веса/категорий, чтобы разные игры настраивали свою систему инвентаря без хардкода.
- **Dialogue/Quest Localization Table** — текстовые строки на языках, привязанные по ключу (иначе Dialogue Graph жёстко зашивает язык).
- **Save Schema / Game State** — описание того, какие переменные/флаги персистентны между уровнями и сессией (нужно фактически для любого жанра, где прогресс переносится).
- **Input Map** — раскладка управления (клавиатура/геймпад/тач), особенно важно если редактор целится в разные жанры (платформер vs top-down vs транспорт требуют разных схем).

### Навигация/AI
- **Path / Spline** — последовательность точек с интерполяцией (linear/bezier), на неё завязаны: патрулирование NPC, движение платформ, траектория камеры в катсцене, рельсы транспорта.
- **NavMesh / Walkable Area** — зона, по которой AI может строить маршрут (актуально для top-down/RPG с свободным AI-перемещением, в отличие от Path, который жёстко фиксирован).
- **AI Behavior Preset** — набор параметров поведения (агрессивный/патрульный/пассивный, радиус обнаружения, скорость) переиспользуемый между разными NPC-Actor'ами.

### Прочее
- **Prefab / Actor Template** — сохранённый Actor с преднастроенными компонентами и параметрами, чтобы не пересобирать врага/дверь/сундук с нуля на каждом уровне. По сути это ядро масштабируемости редактора.
- **Sequence / Cutscene Timeline** — таймлайн, управляющий несколькими Actor'ами/камерой/UI одновременно (диалог + движение камеры + анимация персонажа синхронно). Нужен для катсцен и скриптовых событий.

## 3. Component-типы (навешиваются на Actor, а не отдельные asset-типы)

Это отвечает на твой вопрос "что нужно для диалогов / сбора предметов / влияния на объекты уровня / транспорта" — почти всё это не новый **тип**, а **компонент** на Actor:

| Компонент | Что даёт | Жанры |
|---|---|---|
| **Collider** | форма коллизии (уже упомянута в Actor) | все |
| **Trigger** | зона, реагирующая на вход/выход игрока (уже упомянута) | все |
| **TransformAnimation** | движение/поворот/масштаб по Path или кривой (уже упомянута) | платформер (лифты), transport (рельсы) |
| **SpriteAnimation / UIAnimation** | покадровая анимация (уже упомянута) | все |
| **Interactable** | делает Actor доступным для взаимодействия по кнопке (открыть, поговорить, толкнуть) — с указанием радиуса и подсказки | adventure, RPG, puzzle |
| **Pickup** | ссылка на Item Definition + логика удаления с уровня и добавления в инвентарь | почти все жанры со сбором предметов |
| **Dialogue Trigger** | ссылка на Dialogue Graph, запускается через Interactable или Trigger | RPG, adventure, visual novel |
| **Damage / Health** | HP, урон, смерть, неуязвимость | экшен, платформер, RPG |
| **Movable / Pushable** | Actor можно толкать/тащить (ящики-головоломки) | puzzle, Sokoban-подобные |
| **Mountable / Vehicle Seat** | точка посадки игрока в транспорт, передача управления от игрока к Actor-транспорту и обратно | racing, exploration с транспортом |
| **PathFollower** | Actor (платформа, враг, транспорт) двигается по заданному Path автоматически | все жанры с движущимися объектами |
| **Spawner** | периодически/по условию создаёт другие Actor'ы (враги, предметы, снаряды) | экшен, roguelike |
| **StateMachine / Behavior** | конечный автомат состояний для NPC/механизмов (патруль → погоня → атака) | экшен-AI |
| **PlayerStart** | точка появления игрока на уровне (обычно спец-маркер, не полноценный Actor) | все |
| **Checkpoint / SavePoint** | фиксирует прогресс, точку возрождения | platformer, RPG |
| **Climbable / Ladder** | разрешает вертикальное перемещение вдоль Actor'а | platformer, metroidvania |
| **ConveyorBelt / Zipline / JumpPad / Portal** | частные случаи "изменяет перемещение игрока при контакте" — по сути специализированный `Volume`/Trigger + импульс/телепорт | platformer, puzzle |
| **DestructibleContainer** | сундук/ваза/ящик: при взаимодействии выдаёт Pickup и меняет спрайт/исчезает | adventure, экшен |
| **VariableModifier** | при триггере меняет глобальный флаг/переменную (для завязки на Quest/Dialogue условия) | все жанры с логикой уровня |

## 4. Итоговая рекомендация по приоритету внедрения

**Уровень 1 (без этого не поедет ни один жанр кроме чистого paint-уровня):**
`Sprite Animation Clip`, `Tileset/Tilemap`, `Prefab`, `PlayerStart`, `Path/Spline`, `Interactable`-компонент, `Pickup`-компонент.

**Уровень 2 (открывает RPG/adventure/квесты):**
`Dialogue Graph`, `Item Definition`, `Inventory Schema`, `Quest/Objective`, `VariableModifier`-компонент, `NavMesh`.

**Уровень 3 (полировка и специфичные жанры):**
`Particle Effect`, `Material/Shader Preset`, `Light`, `Parallax Layer`, `Sequence/Cutscene`, `Mountable/Vehicle`-компонент, `AI Behavior Preset`, `Save Schema`, `Localization Table`, `Input Map`.

Такой порядок даёт рабочий 2D level-editor уже на Уровне 1 (платформеры/аркады), Уровень 2 открывает нарративные жанры, Уровень 3 — это уже качество продакшна, а не блокер.
